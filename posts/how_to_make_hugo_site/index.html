How to Make Hugo Site

March 20, 2025 4-minute read

This is a note reviewing how I set up this website. I used Hugo and Github, and chose hugo-coder as the theme. I referred to this Youtube, “Creating a Blog with Hugo and Github in 10 minutes” by Ryan Schachte (https://www.youtube.com/watch?v=LIFvgrRxdt4) and README of the theme hugo-coder (https://github.com/luizdepra/hugo-coder). This note is for future me completely forgotten what I did last night.

Setting Up Hugo Site Link to heading

Downloading Hugo Link to heading

First, download Hugo.
Run brew install hugo on the Terminal app.
I had to install Homebrew at the very beginning to do this step as well. (This is a super easy process, just run: /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)").

Create Github repository Link to heading

Make sure you have a Github account.
Create two repository on your Github. One is for storing the code (named blog or whatever), another is for deploying (named <your-github-username>.github.io).
Make sure both repository are public.

Clone your repository on your computer Link to heading

Run git clone https://github.com/<your-github-username>/blog.git
This create the folder blog.

Create a New Hugo Site. Link to heading

Go inside of blog file and create your site.
Go blog folder: cd blog, and run:
hugo new site <the folder name of your-blog>
cd <the folder name of your-blog>
New Hugo Site is created in the folder on blog folder, <the folder name of your-blog>.

Add a Hugo Theme Link to heading

Variety of Hugo Themes are available on the Hugo Website. Choose one, download, and then set it as your theme on configuration file.

I chose “hugo-coder” (because downloading process was super simple), and here is what I did. The link for hugo-coder: https://github.com/luizdepra/hugo-coder

Download the theme. On the folder <the folder name of your-blog> (use cd and cd ../ to navigate), run this code: git submodule add https://github.com/luizdepra/hugo-coder.git themes/hugo-coder

Set it as your theme: Open configuration file by running nano hugo.toml and make sure theme = "coder". (inside "" could be coder or hugo-coder, or what ever. Important point is inside "" and your file name for theme is the same).
Once you done, close configuration file (by saving change) by control + X > y > Enter.

Check Link to heading

Check if it worked: On the folder <the folder name of your-blog>, run hugo server. This build your website. You can access your site via http://localhost:1313/.
Close this website mode by control + C.

First Post Link to heading

You can create your post very easily. On the folder <the folder name of your blog>, run hugo new posts/<your post title>.md
This creates a new post.

Then, write down and publish your post.
Go to the folder that has your post: cd <the folder name of your blog>/content/posts
Open the markdown file (= your post) to edit: nano <your post title>.md
You can edit title, date, etc, and you can write down any notes by using Markdown below.
Make sure draft = false (= published), not draft = true (= not published) to see it on your website.
Once you done, close markdown file (by saving change) by control + X > y > Enter.

You can check if your post is visible by “Check” (hugo server > access via https://local host:1313/).

Deploy Link to heading

Go back to your normal folder (where you are when you open the terminal app) by using cd ../
Clone your another repository for deploy: git clone https://github.com/<your-github-username>/<your-github-username>.github.io.bit
This creates a folder named <your-github-username>.github.io

Add Readme file Link to heading

Run these one by one (add README on your repository for deploying):
Switch to a new branch main: git checkout -b main
Add Readme file: touch README.me
Check what is going on: git status
Then…,
git add .
git commit -m "adding readme"
git push origin main
This creates Readme file on your github repository (<your-github-username>.github.io).
Running git push origin main for the first time required Github id and password, and I had to create and use token as a password.
main is a branch of your repository.

Enable Deploying Link to heading

Delete public folder on user/blog/<the folder name of your-blog>
(I do not know why, but this process was necessary to follow Youtube instruction, as I already had public file.)
On <the folder name of your-blog>, run git submodule add -b main https://github.com/<your-github-username>/<your-github-username>.github.io.git public
You can check the posts are now visible by “check” (run hugo server on <the foldername of your-blog> > http://localhost:1313/).

On <the folder name of your-blog>, run hugo -t coder (coder should be the folder name of your theme).
Go to public folder (cd public) and look inside (ls): some files should be added (like index.html).

Still on the public folder, check you are on the same repository: git remote -v
If it shows your repository for deploying (not blog, storing one), now you are ready.
See what changes you made: git status (you’ll see files like index.html that are newly created).

Then…,
git add .
git commit -m "init commit"
git push origin main

Now your website should be accessible via link (available on your setting, Github page): https://<your-github-username>.github.io/
